프로젝트 소개

복잡한 서비스 기능을 배제하고, 서비스 소개 랜딩페이지와 CRM 핵심 기능(상품 등록·조회)만을 빠르게 개발·배포하여 SPRAP CRM의 초기 시장 검증을 지원하는 MVP를 구축합니다.

핵심 요구사항 분석

  • 랜딩페이지 모듈 — 6개 섹션 PC·모바일 반응형 퍼블리싱, 디자인 시안 기반 UI 컴포넌트·스타일링, 부드러운 스크롤 및 트랜지션 애니메이션, CTA 클릭 시 회원가입·로그인 페이지 연동
  • 계정 관리 모듈 — 이메일·비밀번호 회원가입, 로그인 및 세션 유지, 입력 폼 유효성 검사 및 오류 메시지, 로그인 성공 시 CRM 대시보드 라우팅
  • 상품 관리 모듈 — 상품명·종목·지역·가격·설명 텍스트 정보 등록 폼, 단일 이미지 업로드 및 미리보기, 상품 목록 조회·상세 보기, 수정·삭제(CRUD)
  • CRM 대시보드 모듈 — 로그인 후 진입하는 메인 대시보드 UI, 메뉴 네비게이션 구성, 미구현 메뉴 "준비 중" 알림, 전체 화면 반응형 레이아웃
  • 비기능 요구사항 — PC·모바일 무손상 반응형, 빠른 로딩 속도, 비밀번호 암호화 저장 및 기본 세션 보안 처리

제안 포인트

1. 디자인 시안 → 픽셀 단위 반응형 퍼블리싱. 제공해 주시는 시안을 기준으로 6개 섹션을 PC·태블릿·모바일 전 구간에서 깨짐 없이 구현하며, 섹션 간 부드러운 스크롤과 진입 트랜지션 애니메이션까지 적용합니다.

2. 서버리스로 빠르고 안전하게. Supabase(또는 Firebase)를 활용해 별도 서버 운영 부담 없이 이메일 인증·DB·이미지 스토리지를 통합합니다. 비밀번호는 플랫폼 표준 해시 방식으로 암호화 저장되고, Row Level Security 기반으로 데이터 접근을 통제합니다.

3. MVP에 최적화된 확장 가능한 구조. 상품 관리 외 메뉴는 "준비 중" 처리하되, 이후 결제·예약 등 기능 확장이 용이하도록 모듈화된 컴포넌트·라우팅 구조로 설계합니다.

4. 명확한 인수인계. 배포 URL·전체 소스코드·테스트 계정·주요 화면 캡처·실행 안내 자료를 체계적으로 정리하여, 개발 완료 후 내부에서 원활히 운영할 수 있도록 지원합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

SPRAP CRM의 초기 시장 검증을 위해, 복잡한 기능(결제·예약·정산 등)을 배제하고 서비스 소개용 랜딩페이지와 내부 관리용 CRM의 핵심 기능(상품 등록·조회)만을 빠르게 개발·배포하는 것을 목표로 합니다. 의뢰자가 제공하는 디자인 시안에 맞추어 반응형 웹을 퍼블리싱하고, 필수 로그인 및 상품 관리 기능을 구현합니다.

2. 사용자 정의

사용자 유형설명주요 기능
방문자 (비로그인)랜딩페이지를 통해 서비스를 인지하는 잠재 고객랜딩페이지 열람, 회원가입·로그인 진입
CRM 사용자 (로그인)로그인 후 내부 관리 기능을 사용하는 운영 담당자대시보드 진입, 상품 등록·조회·수정·삭제

3. 기능 요구사항

3.1 랜딩페이지 모듈

  • [필수] 6개 섹션 PC·모바일 반응형 퍼블리싱 (디자인 시안 기반)
  • [필수] 섹션 간 부드러운 스크롤 및 기본 트랜지션 애니메이션
  • [필수] 버튼 클릭 시 회원가입·로그인 페이지 이동 연동

3.2 계정 관리 모듈

  • [필수] 이메일·비밀번호 회원가입
  • [필수] 이메일 로그인 및 사용자 세션 유지
  • [필수] 입력 폼 유효성 검사 및 오류 메시지 출력
  • [필수] 로그인 성공 시 CRM 메인 대시보드 라우팅

3.3 상품 관리 모듈

  • [필수] 상품명·종목·지역·가격·설명 텍스트 정보 등록 폼
  • [필수] 단일 이미지 업로드 및 미리보기
  • [필수] 상품 목록 조회 및 상세 보기
  • [필수] 등록 상품 정보 수정 및 삭제

3.4 CRM 대시보드 모듈

  • [필수] 로그인 후 진입 메인 대시보드 UI
  • [필수] 상품 관리 외 메뉴 네비게이션 구성
  • [필수] 미구현 메뉴 클릭 시 "준비 중" 알림 출력
  • [필수] 전체 화면 반응형 레이아웃

4. 비기능 요구사항

항목요구사항
반응형PC·모바일 환경에서 깨짐 없이 동작하는 반응형 웹
성능코드 스플리팅·이미지 최적화로 빠른 초기 로딩 속도 보장
보안비밀번호 암호화 저장, 기본 세션 보안 처리, DB 접근 보안 룰(RLS) 적용

5. 기술 제약사항

프론트엔드는 React.js(또는 Vue.js)로 구현하며, 본 제안에서는 React + Vite + TypeScript + Tailwind CSS를 권장합니다. 백엔드는 Supabase 또는 Firebase 등 서버리스 아키텍처를 활용하며, 본 제안에서는 이메일 인증·Postgres·이미지 스토리지·RLS 보안을 통합 제공하는 Supabase를 권장합니다. 배포는 Vercel을 기준으로 합니다. (의뢰자 선호에 따라 협의 조정 가능)

6. 범위 외 사항 (Out of Scope)

  • 결제, 예약, 정산, 채팅, 알림, 리뷰, 쿠폰, 포인트 등 실제 서비스 기능
  • 본인인증, 소셜 로그인
  • 네이티브 앱 개발
  • 디자인 시안 제작 (의뢰자 측 제공)
  • 장기 유지보수 계약
